Getting started

When your application is approved you should receive an email with account details for the Hadoop service at SARA. You have also been added to your own private Github repository, and have received details on this as well. You can use this information within a virtual machine image we have prepared for you. Carefully read the below documentation on how to get started.

Setting up the virtual machine environment

Prerequisites

The image is an OVA (Open Virtualization Archive) file which should work in at least VirtualBox and VMWare. It was tested under VirtualBox on Linux and Windows hosts. Because of this we advise you to use VirtualBox.

Download

The VM image is called "HadoopVM.ova" and can be downloaded from the SARA FTP server. The image is 2.7 GB in size.

Run

Once you have downloaded the image you can import it into VirtualBox by selecting "File → Import Appliance". The dialogs should be straight forward. Depending on your own machine you may want to tweak some of the VM settings such as total memory and the number of processors.

Additional image information

On the image we have installed Ubuntu GNU/Linux 12.04. Among other, we installed the following applications:

  • Oracle JDK 1.6
  • Hadoop 0.20.205
  • Pig 0.10.0
  • Eclipse 3.7.2

The user account is called 'participant' and has the password 'hadoop'. You need this password for tasks that require administrative rights (such as installing additional software).

Running the Common Crawl examples

The system comes with Eclipse and a modified version of the Common Crawl examples. When you launch Eclipse for the first time it will show the project README.md with instructions on running the examples.

Developing your own Hadoop application

We recommend that you use the Common Crawl examples as a starting point for your own Hadoop application. You should also use their helper classes such as the ArcRecordReader.

Here is a list of Hadoop-related resources that can help you:

  • The API documentation of Hadoop.
  • Search-Hadoop.com, a Hadoop-specific search engine.

When you run into problems the Common Crawl Group or the Hadoop user mailing list might be able to assist you.

Using SARA’s Hadoop cluster Alley

Restrictions

Note the fair use policy when submitting the cluster:

  • You can only do a total of two runs on the full dataset.
  • You may do an unlimited number of runs on the small dataset (just not simultaneously).

Because the Hadoop cluster is a multi-tenant system your job might not be scheduled for immediate execution. Do not wait with your final runs until a few days before the deadline! Even more so since you also need to report your results before the deadline.

Authentication

Before you can access the cluster you need to authenticate yourself with

$ kinit LOGIN

where LOGIN is the login name we have send you by email. The alias hadoop-alley functions like the hadoop command, but uses the cluster configuration.

Job submission

To submit a job to the cluster you use

$ hadoop-alley jar JARFILE MAINCLASS ARGUMENTS

(The hadoop-alley command is an alias for the plain hadoop command that sets the configuration directory for the cluster).

Job/cluster monitoring

Both the namenode and jobtracker have a web interface. The Firefox browser in the VM image already contains bookmarks to these pages. You also need to authenticate with kinit to get access to the web interfaces.
