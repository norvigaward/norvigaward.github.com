Getting started

When your application is approved you should receive an email with account details for the Hadoop service at SARA. You have also been added to your own private repository, and have received details on this as well. You can use this information within a Virtual Machine image we have prepared for you. Carefully read the below documentation on how to get started.

Setting up the virtual machine environment

Prerequisites

The image is an OVA (Open Virtualization Archive) file which should work in at least VirtualBox and VMWare. It was tested under VirtualBox on Linux and Windows. Because of this we advise you to use VirtualBox.

Download

The VM image can be downloaded from the SARA FTP server. The image is 2.7 GB in size.

Run

Once you have downloaded the image you can import it into VirtualBox by selecting "File -⟨ Import Appliance". The dialogs should be straight forward. Depending on your own machine you may want to tweak some of the VM settings such as total memory and the number of processors.

Running the Common Crawl examples

The system comes with Eclipse and a modified version of the common crawl examples. When you launch Eclipse for the first time it will show the project README with instructions on running the examples.

Developing your own Hadoop application

Using the Alley Hadoop cluster

Restrictions

Note the restrictions when submitting the cluster:

  • You cannot run more than one job simultaneously.
  • You can only do a total of two runs on the full dataset.
  • You may do an unlimited number of runs on the small dataset (just not simultaneously).

Authentication

Before you can access the cluster you need to authenticate yourself with

$ kinit LOGIN

where LOGIN is the login name we have send you by e-mail. The alias hadoop-alley functions like the hadoop command, but uses the cluster configuration.

Job submission

To submit a job to the cluster you use

$ hadoop-alley jar JARFILE MAINCLASS ARGUMENTS

(The hadoop-alley command is an alias for the plain hadoop command that sets the configuration directory for the cluster).

Job/cluster monitoring

Both the namenode and jobtracker have a web interface. The Firefox browser in the VM image already contains bookmarks to these pages. You also need to authenticate with kinit to get access to the web interfaces.
